# Рівні загроз

🟢 Зелений → 🟡 Жовтий → 🟠 Помаранчевий → 🔴 Червоний

# Каскадне зниження (downgrade)

Стейт-машина, стан у PostgreSQL (таблиця `threat_state`), не в пам'яті процесу.

- T+1хв після нейтралізації з 🔴 → 🟠
- T+6хв → 🟡
- T+21хв → 🟢
- Абсолютне правило переривання: новий тригер вищого рівня скидає всі таймери
- "Локаційно втрачена ціль" (`is_locationally_lost`) — примусовий downgrade на один крок після 20 хв без оновлення локації

# Дедуплікація

`dedup:{hash}` в Redis, TTL 30 сек, hash = SHA-256(нормалізований текст) + `threat_type` + `target_location`. Семантичне зіставлення (перефразовані дублі з різних каналів) — через поля `target_location`+`threat_type` з JSON-відповіді LLM, не по сирому тексту. Лічильник підтверджень з 2+ джерел — тригер для "чисто"/"відбій".
