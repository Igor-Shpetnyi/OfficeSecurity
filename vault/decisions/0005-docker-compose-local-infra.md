# ADR-0005: Docker Compose для інфраструктури локальної розробки

**Контекст:** ADR раніше (backlog, обговорення контейнеризації) залишало Docker поза застосунком — юзербот і admin panel лишаються на systemd/venv (аргумент: живий дебаг reconnect-логіки Telethon). Але цей аргумент не стосується Postgres/Redis — їх не дебажать інтерактивно, їх або піднімають, або ні.

**Рішення:** `docker-compose.yml` у корені репозиторію піднімає Postgres 16 і Redis 7 для **локальної розробки**. Схема (`app/db/schema.sql`) змонтована в `/docker-entrypoint-initdb.d/` — застосовується автоматично при першому старті контейнера з порожнім volume.

**Явно поза межами цього рішення:** де і як стоятимуть Postgres/Redis на проді (та сама контейнеризація на Hetzner, чи нативний `apt install`, чи керований DB-сервіс) — окреме питання, відкладене до обговорення деплою. Застосунок (юзербот, admin panel) як і раніше не контейнеризується цим ADR.

**Чому саме зараз:** без цього кожен, хто піднімає проєкт локально, має вручну ставити Postgres/Redis з системних пакетів, стежити за версіями, вручну накатувати `schema.sql`. `docker-compose up` дає відтворюване середовище в одній команді.

**Статус:** accepted, обмежено локальною розробкою.
