# ADR-0006: Дизайн-система для адмін-панелі

**Контекст:** користувач надав `vault/design/design.md` — дизайн-систему проєкту **Horyzont** (Modern Dark Dashboard: CSS custom properties, `data-theme` перемикання, Inter, Lucide-стиль іконок), з вимогою використовувати її як єдиний дизайн-преференс для дашборду OfficeSecurity.

**Нюанс:** документ описує React-реалізацію (`ThemeProvider`, `useTheme`, `.tsx`-компоненти) — це деталі стеку Horyzont, а не самого дизайну. ТЗ OfficeSecurity (розділ 11) явно відмовився від React для адмін-панелі: "не варто городити React заради однієї сторінки з кнопками рівня і формою причини".

**Рішення:** відтворити дизайн-систему (кольори, типографіка, layout, компоненти, анімації, тема) **один-в-один** через ту саму техніку — CSS custom properties + атрибут `data-theme` на `<html>` — але без React: чистий CSS (`app/admin/static/css/theme.css`) + ванільний JS для перемикання теми (`app/admin/static/js/theme.js`) поверх Jinja2-шаблонів. Anti-flash-скрипт із §8 design.md працює однаково незалежно від фреймворку.

**Навігація й контент — не з Horyzont.** Пункти меню, картки, таблиці — під домен OfficeSecurity (Дашборд/Канали/Події), а не контакти/групи/словники з оригінального документа. Дизайн-система (§2-§7 design.md) застосовується буквально; §9 (мапінг секцій під Horyzont) і React-специфічні частини §8 — ні.

**Іконки:** без `lucide-react`/npm — inline SVG в тому ж візуальному стилі (stroke-based, 24×24, `currentColor`), щоб не тягнути Node.js-залежність у Python-проєкт.

**Статус:** accepted. Правило зафіксовано в [CLAUDE.md](../../CLAUDE.md) — будь-який подальший UI в цьому проєкті використовує виключно цю дизайн-систему.
